# 🌙 Dreamweaver — Bedtime Story Generator

A front-end-only **Next.js** app that turns a child's real day into a
personalized bedtime story, written in the theme of your choice and generated by
an AI model through the **OpenRouter API**.

> Built for **Coded Class** — Stages 1–3.

## What it does

- Enter the child's **name**, **age**, pick a **theme**
  (Fantasy 🧚 / Comedy 🤡 / Sci-Fi 🚀 / Spooky 👻), and describe **their day**.
- Sends a request to OpenRouter and **streams the story back** live, word by word.
- Default model: `meta-llama/llama-3.1-8b-instruct:free`.

## Where the API key lives (read this!)

For the class exercise, the key is a **variable at the top of `lib/config.ts`**:

```ts
export const OPENROUTER_API_KEY =
  process.env.NEXT_PUBLIC_OPENROUTER_API_KEY || "PASTE_YOUR_OPENROUTER_KEY_HERE";
```

**Local use:** either paste your key directly into that string, **or** create a
file named `.env.local` with:

```
NEXT_PUBLIC_OPENROUTER_API_KEY=sk-or-your-key-here
```

Get a free key at <https://openrouter.ai/keys>.

⚠️ **Security note (part of the assignment):** this is a front-end app, so the
key is visible to anyone who reads the page source. That's acceptable for a
*free* model key during the class. When you **push to GitHub + deploy to
Vercel**, do **not** hardcode your key — use the `.env.local` / Vercel
environment variable instead so it isn't committed to a public repo.

## Run it locally

```bash
npm install
npm run dev
```

Open <http://localhost:3000>.

## Deploy to Vercel

1. Push this folder to GitHub.
2. Import the repo at <https://vercel.com/new>.
3. In **Project Settings → Environment Variables**, add
   `NEXT_PUBLIC_OPENROUTER_API_KEY` = your key.
4. Deploy.

## How the rubric is met

| Requirement | Where |
| --- | --- |
| Real OpenRouter request + displays response | `lib/openrouter.ts` (streaming) |
| Key stored as a variable at top of file | `lib/config.ts` |
| Required model `llama-3.1-8b-instruct:free` | `lib/config.ts` → `DEFAULT_MODEL` |
| System prompt + personality ("Dreamweaver") | `lib/openrouter.ts` → `buildSystemPrompt` |
| UI matches concept (title, inputs, output) | `app/page.tsx` |
| Stage 3 — Loading state | streaming + "weaving…" indicator |
| Stage 3 — Error state | caught + shown in a red banner |
| Stage 3 — Copy to clipboard | "Copy" button on the story |
| Stage 3 — Improve the UI | night-sky theme, glass cards |
| Stage 3 — Creativity slider | temperature slider (0.0–1.0) |
| Stage 3 — Model switcher | dropdown of free models |

## Project layout

- `lib/config.ts` — API key + model list (edit this).
- `lib/openrouter.ts` — builds the prompt and streams from OpenRouter.
- `lib/themes.ts` — the four themes and their prompt guidance.
- `app/page.tsx` — all UI and form state.
- `components/StarrySky.tsx` — the animated stars + moon.

## Safety

Prompts keep every story age-appropriate — even "Spooky" is friendly-spooky,
never frightening. Stories are AI-generated, so a parent should always read along.
