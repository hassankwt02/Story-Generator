# 🌙 Dreamweaver — Bedtime Story Generator

A **Next.js** app that turns a child's real day into a personalized bedtime
story, written in the theme of your choice and generated by an AI model through
the **OpenRouter API**.

> Built for **Coded Class** — Stages 1–3 (plus the ElevenLabs Extra Mile).

## Architecture (and why the API key is safe)

The UI is front-end, but API calls go through small **server-side Next.js API
routes** so the keys never reach the browser:

```
Browser  →  /api/generate  (server, holds OPENROUTER_API_KEY)  →  OpenRouter
Browser  →  /api/narrate   (server, holds ELEVENLABS_API_KEY)  →  ElevenLabs
```

Keys are read from `process.env` using **plain (non-`NEXT_PUBLIC_`) variables**,
so they are *not* inlined into the browser bundle. (An earlier version used
`NEXT_PUBLIC_…` and the key was visible in the page source — this server-route
version fixes that.)

## What it does

- Enter the child's **name**, **age**, pick a **theme**
  (Fantasy 🧚 / Comedy 🤡 / Sci-Fi 🚀 / Spooky 👻), and describe **their day**.
- Sends a request to OpenRouter and **streams the story back** live, word by word.
- Default model: `google/gemma-4-31b-it:free` (the assignment's
  `llama-3.1-8b-instruct:free` was discontinued by OpenRouter).
- 🔊 **Read aloud** — narrate the finished story with a deep, warm storyteller
  voice via **ElevenLabs** (Extra Mile challenge).

## 🔊 Read aloud (ElevenLabs) & a note on "Morgan Freeman's voice"

The finished story can be narrated out loud. This uses the ElevenLabs
text-to-speech API (server-side, via `/api/narrate`) and needs a **separate**
ElevenLabs key — add `ELEVENLABS_API_KEY` to `.env.local`. The "Read aloud"
button only appears when that key is configured.

This started as a request for "Morgan Freeman's voice." We deliberately **do not**
clone Morgan Freeman (or any real person): cloning an identifiable person's voice
without consent violates ElevenLabs' terms and that person's voice-likeness
rights. Instead we use a **deep, warm narrator voice from ElevenLabs' standard
library** that evokes the same soothing bedtime-storyteller feel. The voice is
configurable in `lib/config.ts` (`NARRATOR_VOICE_ID`) — swap in any voice you
have the rights to use.

## Where the API keys live

Create a file named `.env.local` in the project root (it is gitignored, so it is
never committed):

```
OPENROUTER_API_KEY=sk-or-your-key-here
ELEVENLABS_API_KEY=your-elevenlabs-key   # optional, only for "Read aloud"
```

Note: **no `NEXT_PUBLIC_` prefix** — that keeps the keys server-only so they are
never sent to the browser. Get a free OpenRouter key at
<https://openrouter.ai/keys>. After editing `.env.local`, **restart the dev
server** so the new values load.

You can verify the key isn't exposed: run `npm run build`, then
`grep -r "sk-or" .next/static` — it returns nothing (the key is only in the
server runtime, never the browser bundle).

## Run it locally

```bash
npm install
npm run dev
```

Open <http://localhost:3000>.

## Deploy to Vercel

1. Push this folder to GitHub.
2. Import the repo at <https://vercel.com/new>.
3. In **Project Settings → Environment Variables**, add
   `OPENROUTER_API_KEY` = your key (and `ELEVENLABS_API_KEY` if you want
   narration). No `NEXT_PUBLIC_` prefix — they stay server-side.
4. Deploy.

## How the rubric is met

| Requirement | Where |
| --- | --- |
| Real OpenRouter request + displays response | `app/api/generate` + `lib/openrouter.ts` (streaming) |
| Key kept secret (server-side, not in browser) | `.env.local` + `app/api/*` |
| Required model (note: discontinued upstream) | `lib/config.ts` → `DEFAULT_MODEL` |
| System prompt + personality ("Dreamweaver") | `lib/story.ts` → `buildSystemPrompt` |
| UI matches concept (title, inputs, output) | `app/page.tsx` |
| Stage 3 — Loading state | streaming + "weaving…" indicator |
| Stage 3 — Error state | caught + shown in a red banner |
| Stage 3 — Copy to clipboard | "Copy" button on the story |
| Stage 3 — Improve the UI | night-sky theme, glass cards |
| Stage 3 — Creativity slider | temperature slider (0.0–1.0) |
| Stage 3 — Model switcher | dropdown of free models |
| Extra Mile — ElevenLabs voice output | `lib/elevenlabs.ts` + "Read aloud" button |

## Project layout

- `app/api/generate/route.ts` — server route that holds the OpenRouter key and
  streams the story.
- `app/api/narrate/route.ts` — server route that holds the ElevenLabs key and
  returns narration audio.
- `app/api/status/route.ts` — reports whether each key is configured (no key
  values returned) to drive the UI.
- `lib/story.ts` — the system/user prompt builders (shared, no secrets).
- `lib/openrouter.ts` / `lib/elevenlabs.ts` — thin client fetchers that call the
  routes above.
- `lib/config.ts` — non-secret config (model list, narrator voice id).
- `lib/themes.ts` — the four themes and their prompt guidance.
- `app/page.tsx` — all UI and form state.
- `components/StarrySky.tsx` — the animated stars + moon.

## Safety

Prompts keep every story age-appropriate — even "Spooky" is friendly-spooky,
never frightening. Stories are AI-generated, so a parent should always read along.
